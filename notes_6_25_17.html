Client-side code

Options for creating a web app

What are Frameworks?

  • frameworks provide pre-made functionality and patterns
  • framework code + your app code = working web apps

Ajax : Asynchronous JavaScript and XML

Gmail is first big site to use AJAX

Single Page App(SPA)

Server sends one bare-bones HTML file and lots of JS to the browser. Pro. Organizes client code in structures fashion Rich Interactions : Updates the screen immediately instead of waiting for request-response cycle from the server. Frameworks : Backbone(2010), Angular(2012), React(2013) Con: Not SEO friendly (SEO works on HTML tags number)

SPA+API

API provides the data. SPA renders the content. SPA is JavaScript; API can be written in a variety of languages. Pro: One API can be used by a variety of clients. Con: multiple code bases, often times in multiple languages.

SAP+Cloud Database

Cloud Database provides the data(eg. Firebase). SPA renders the content. "Serverless" web app. Pro:Quick to implement. Con: Limited number of choices for client-side cloud databases.

SPA+Server-Side Rendering

Run SPA code on the server to create HTML on initial page load. Use client side code for subsequent page updates. Pro: SEO friendly. Faster initial page load. Con: Takes time to set up server-side rendering.

Intro to Git Workflows

Why Version Control

View history of all code changes. Allow multiple people to work on project simultaneously. Experiment with new code without breaking existing code. Never lose code. Look at previous versions of the code to find out when a bug was introduced.

Basic Commands : Local Repo

  • git branch (name of branch) : create branch
  • git checkout (name of branch) : switch the branch
  • git checkout -b (name of branch) : create new branch and switch to it
  • git branch : list branches
  • git merge : combine two branches

Intro to Github

  • Github stores repos

  • git remote add origin (server) : connect local repo to remote repo
  • git pull: get latest changes from remote server
  • git push (remote) (branch)
